## 1. Project Overview

### Task
5-class Diabetic Retinopathy grading from retinal fundus images:

| Grade | Class | Clinical Description |
|-------|-------|----------------------|
| 0 | No DR | No visible retinal changes |
| 1 | Mild NPDR | Microaneurysms only |
| 2 | Moderate NPDR | More than microaneurysms, less than severe |
| 3 | Severe NPDR | Extensive hemorrhages, venous beading |
| 4 | Proliferative DR | Neovascularization — risk of blindness |

### Key Design Decisions

| Design Choice | Reason |
|---------------|--------|
| Custom CNN (not pretrained) | Full architectural control; avoids ImageNet-domain mismatch |
| SE-Block attention | Channel recalibration highlights diagnostic lesion-relevant features |
| Focal Loss | Down-weights easy examples; forces focus on hard, ambiguous DR grades |
| K-Fold Cross Validation | Maximizes use of 400 samples; more reliable generalization estimate |
| Class-weighted loss | Ensures Severe/Proliferative classes are not ignored |
| Grad-CAM | Provides visual explanation of model decisions for clinical trust |
| 256×256 input | Balance between lesion detail and compute efficiency |

## 2. Architecture

```
DRNet Architecture
──────────────────────────────────────────────────────────────────
Input            (B, 3, 256, 256)
│
├─ Stem Block    7×7 Conv → BN → ReLU → 3×3 Conv → BN → ReLU → MaxPool
│                Output: (B, 64, 64, 64)
│
├─ Stage 1       ConvBlock(64→64,  stride=1) + SE Block + Residual
│                Output: (B, 64, 64, 64)
│
├─ Stage 2       ConvBlock(64→128, stride=2) + SE Block + Residual
│                Output: (B, 128, 32, 32)
│
├─ Stage 3       ConvBlock(128→256, stride=2) + SE Block + Residual
│                Output: (B, 256, 16, 16)
│
├─ Stage 4       ConvBlock(256→512, stride=2) + SE Block + Residual
│                Output: (B, 512, 8, 8)   ← Grad-CAM target
│
├─ GAP           AdaptiveAvgPool2d(1)
│                Output: (B, 512, 1, 1)
│
└─ Classifier    Linear(512→256) → ReLU → Dropout(0.5) → Linear(256→5)
                 Output: (B, 5) logits
──────────────────────────────────────────────────────────────────
Trainable Parameters: ~3.1M
```

### SE-Block Detail

```
Input (B, C, H, W)
  → Squeeze: GlobalAvgPool → (B, C)
  → Excitation: FC(C→C/16) → ReLU → FC(C/16→C) → Sigmoid
  → Scale: input × sigmoid_weights (broadcast over H, W)
Output (B, C, H, W)  ← channel-recalibrated feature map
```

## 6. Training

### Quick Start

```bash
python src/train.py
```

This uses default settings: 5-fold CV, 80 epochs, batch size 16, Focal Loss, AdamW.

### Custom Training

```bash
python src/train.py \
    --image_dir data/images \
    --csv_path data/labels.csv \
    --epochs 100 \
    --batch_size 16 \
    --n_folds 5 \
    --lr 1e-3 \
    --loss_type focal \
    --seed 42
```

### Using a JSON config

```bash
# Create config.json:
{
  "image_dir": "data/images",
  "csv_path":  "data/labels.csv",
  "epochs":    100,
  "n_folds":   5,
  "batch_size": 16,
  "lr": 0.001,
  "loss_type": "focal",
  "focal_gamma": 2.0,
  "patience": 20

python src/train.py --config config.json
```

## 8. Grad-CAM Explainability

### Single image

```bash

    --checkpoint outputs/checkpoints/best_fold1.pth \
    --image data/images/im042.jpg \
    --true_label 2 \
    --output_dir outputs/gradcam
```

Generates a 4-panel figure:
1. Original fundus image
2. Standalone heatmap
3. Heatmap overlaid on image
4. Per-class confidence bar chart

### Programmatic usage

```python
from src.gradcam import explain_image

result = explain_image(
    image_path="data/images/im042.jpg",
    checkpoint_path="outputs/checkpoints/best_fold1.pth",
    true_label=2,

print(result["probabilities"])     # [0.03, 0.08, 0.71, 0.12, 0.06]
```

## 11. Research Notes

### Why K-Fold Cross Validation?
With 400 samples, a single train/val split (e.g., 320/80) is highly sensitive to which samples fall in the validation set. K-Fold ensures every sample is in the validation set exactly once, giving a much more stable estimate of true generalization performance.

### Why Focal Loss over Cross Entropy?
Standard CE loss assigns equal weight to easy and hard examples. With fundus images, mild/moderate DR grades are visually similar, creating many hard examples that CE treats identically to easy ones. Focal Loss (γ=2) exponentially down-weights easy correct predictions, forcing the optimizer to allocate gradient budget to hard cases — exactly the misclassifications that matter clinically.

### Why SE-Block?
Retinal pathology is distributed — lesions appear in different spatial locations but in specific channel activations. SE blocks perform channel-wise recalibration: they learn which feature map channels (each representing a distinct visual pattern) matter most for the current input. This adds negligible parameters (~0.3% increase) but measurably improves sensitivity to subtle lesion features.

### Sensitivity vs Specificity Tradeoff
In DR screening, **false negatives (missing DR) are far more dangerous than false positives** (unnecessary referrals). The system is designed to maximize sensitivity at the cost of some specificity. The Focal Loss + class weighting combination pushes the model toward this operating point.

## 12. Troubleshooting

| Issue | Solution |
|-------|----------|
| `CUDA out of memory` | Reduce `batch_size` to 8 |
| `FileNotFoundError: labels.csv` | Run `prepare_labels.py --mode template` first |
| `KeyError: 'filename'` | CSV must have exactly `filename` and `label` column headers |
| Low sensitivity on Severe/Proliferative | Increase `focal_gamma` to 3.0 or increase patience |
| Training loss not decreasing | Lower `lr` to `5e-4`; check if images are loading correctly |
| Grad-CAM heatmap is uniform | Ensure `model.eval()` is called; verify checkpoint loads correctly |
| Streamlit import error | Run `pip install streamlit>=1.28.0` |
